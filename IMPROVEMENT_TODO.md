# A股量化系统 - 待改进清单

> 创建时间：2026-02-28 12:02
> 来源：Phase 2 & Phase 3执行中发现的问题
> 优先级：按影响程度排序

---

## 🔴 P0：核心风险（必须解决）

### 1. 过拟合风险 ⚠️
**问题描述**：
- 股票样本量过小（54只）
- 因子数量相对样本量占比过高
- 回测结果可能不具代表性

**影响**：
- 实盘效果可能远低于回测效果
- 策略在新数据上失效

**改进方案**：
- [ ] 扩充股票池至200-300只（中证500或沪深300全成分）
- [ ] 进行样本外验证（split: 70%/30%）
- [ ] 增加滚动窗口回测
- [ ] 使用更多时间跨度数据（7-10年）

**预计工作量**：4-6小时
**责任人**：研究员 + 架构师

---

### 2. 交易成本未考虑 💰
**问题描述**：
- 回测未计算印花税、佣金、滑点
- 频繁调仓成本可能吃掉收益

**影响**：
- 实际净收益可能低于回测收益
- 月度调仓成本约0.5-1%/年

**改进方案**：
- [ ] 在回测中添加交易成本模块
  - 印花税：0.1%（卖出）
  - 佣金：0.03%（双向）
  - 滑点：0.1%（双向）
- [ ] 计算成本后重新回测
- [ ] 对比成本前后的收益差异

**预计工作量**：1-2小时
**责任人**：架构师

---

### 3. 因子有效性不足 📉
**问题描述**：
- 有效因子仅1个（得分≥60分）
- 技术因子因数据缺失导致IC为NaN
- 因子预测力整体偏弱

**影响**：
- 选股策略缺乏稳健性
- 收益主要依赖市场Beta而非Alpha

**改进方案**：
- [ ] 扩展因子库到50-100个候选因子
- [ ] 接入更多数据源（Tushare、聚宽等）
- [ ] 加入机器学习因子（非线性组合）
- [ ] 实现因子动态权重（根据IC变化调整）

**预计工作量**：8-12小时
**责任人**：研究员

---

## 🟡 P1：重要优化（应尽快解决）

### 4. 行业集中风险 🏢
**问题描述**：
- 选股结果行业集中度高（半导体、软件占比大）
- 风格暴露明显

**影响**：
- 行业轮动时策略失效
- 无法真正分散风险

**改进方案**：
- [ ] 实现真正的行业中性化选股（每个行业最多选2只）
- [ ] 加入风格因子约束（市值、估值、成长风格中性）
- [ ] 行业配置约束（单一行业占比≤20%）
- [ ] 风格中性优化器（二次规划）

**预计工作量**：3-4小时
**责任人**：研究员

---

### 5. 收益率差距过大 📊
**问题描述**：
- 目标年化：15%
- 实际回测：5.78%
- 差距：9.22%

**影响**：
- 无法达到盈利目标
- 策略竞争力不足

**改进方案**：
- [ ] 尝试更多因子组合
- [ ] 调整加权策略（尝试等权、IC加权、优化权重）
- [ ] 加入择时模块（市场环境判断）
- [ ] 增加选股数量（Top 10 → Top 20）

**预计工作量**：2-3小时
**责任人**：研究员 + 架构师

---

### 6. 风险控制形同虚设 🛡️
**问题描述**：
- 风险控制只是警告机制
- 未实际执行止盈止损
- 回撤控制未生效

**影响**：
- 无法有效控制下行风险
- 遇到极端行情时损失扩大

**改进方案**：
- [ ] 实现真正的止盈（单股+20%清仓）
- [ ] 实现真正的止损（单股-10%清仓 / 组合-15%）
- [ ] 动态仓位管理（根据市场波动调整）
- [ ] 加入波动率目标（控制组合波动率≤15%）

**预计工作量**：2-3小时
**责任人**：架构师

---

## 🟢 P2：性能优化（有时间再做）

### 7. 数据质量问题 📋
**问题描述**：
- 部分股票数据缺失
- 财务数据（PE_TTM、PB）为占位符

**影响**：
- 因子计算不准确
- 选股结果不可靠

**改进方案**：
- [ ] 接入Tushare获取完整财务数据
- [ ] 实现数据缺失值插值
- [ ] 数据质量自动检验模块
- [ ] 异常值检测和处理

**预计工作量**：4-6小时
**责任人**：研究员

---

### 8. 回测速度慢 ⚡
**问题描述**：
- 54只股票回测约2小时
- 扩充到200只预计8-10小时

**影响**：
- 调试和优化效率低
- 无法快速迭代

**改进方案**：
- [ ] 使用向量化计算替代循环
- [ ] 并行化计算（多进程）
- [ ] 增量回测（只计算新增月份）
- [ ] 缓存中间结果

**预计工作量**：2-3小时
**责任人**：架构师

---

### 9. 飞书推送频率过高 📱
**问题描述**：
- 每日推送可能过于频繁
- 非交易日也会推送

**影响**：
- 信息过载
- 可能打扰正常工作

**改进方案**：
- [ ] 检查交易日历（仅交易日推送）
- [ ] 支持推送频率配置（每日/每周）
- [ ] 重要事件才推送（持仓变化＞50%）
- [ ] 支持推送消息模板选择

**预计工作量**：1小时
**责任人**：创作者

---

### 10. 系统监控缺失 👁️
**问题描述**：
- 无法实时监控策略运行状态
- 异常情况无法及时发现

**影响**：
- 系统故障无法及时处理
- 数据更新失败不知情

**改进方案**：
- [ ] 添加心跳检测
- [ ] 数据更新监控
- [ ] 异常告警通知
- [ ] 性能指标追踪（CPU、内存、运行时间）

**预计工作量**：2-3小时
**责任人**：架构师
