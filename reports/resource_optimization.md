# 资源优化方案

## 1. 问题分析

### 当前系统负载问题

1. **Agent 过度参与日常任务**
   - 每日推送、数据更新等确定性任务都通过 Agent 执行
   - Agent 启动需要加载完整上下文（~2-5秒）
   - 单次 Agent 调用消耗大量内存（~500MB-1GB）

2. **资源浪费**
   - 简单的定时任务不需要 AI 决策
   - 并发执行多个 Agent 可能超过系统内存限制
   - 频繁的 Agent 调用增加系统负载

3. **网络开销**
   - 每个 Agent 调用都需要与主进程通信
   - 不必要的上下文传输

### 3.2 限流控制

#### 并发任务限制

| 级别 | 最大并发任务 | 说明 |
|-----|------------|------|
| **Total** | 2 | 同时最多运行 2 个任务 |
| **Heavy** | 1 | 高负载任务互斥 |

#### 任务优先级调度

```
Critical (关键)
  ↓ 超级优先
High (高)
  ↓ 优先
Medium (中)
  ↓ 正常
Low (低)
  ↓ 尽力而为
```

#### 资源等待队列

```python
class TaskQueue:
    """
    任务队列实现
    - 按优先级排序
    - 资源检查后执行
    - 背压机制防止过载
    """

    def __init__(self, max_concurrent=2, max_heavy=1):
        self.queue = PriorityQueue()
        self.running = {}
        self.max_concurrent = max_concurrent
        self.max_heavy = max_heavy

    def add_task(self, task):
        """添加任务到队列"""
        # 检查资源可用性
        if self._can_run(task):
            self._execute(task)
        else:
            self.queue.put(task)

    def _can_run(self, task):
        """检查是否可以运行任务"""
        # 检查总并发数
        if len(self.running) >= self.max_concurrent:
            return False

        # 检查高负载任务互斥
        if task.profile == 'heavy' and self._count_heavy() > 0:
            return False

        return True
```

### 3.3 内存管理

#### 资源释放策略

1. **即时释放**
   ```python
   def run_task(task):
       try:
           result = execute(task)
           return result
       finally:
           # 显式释放资源
           gc.collect()
           del result
   ```

2. **进程隔离**
   ```bash
   # 使用独立进程执行高负载任务
   python3 run_backtest.py &
   PID=$!
   # 父进程继续执行
   wait $PID
   ```

3. **限制内存使用**
   ```python
   import resource
   resource.setrlimit(resource.RLIMIT_AS, (2 * 1024**3, 2 * 1024**3))
   ```

import psutil

def check_memory_limit(max_mb=1024):
    """检查内存使用，超限则警告"""
    process = psutil.Process()
    memory_mb = process.memory_info().rss / (1024**2)

    if memory_mb > max_mb:
        logger.warning(f"内存使用过高: {memory_mb:.0f}MB > {max_mb}MB")
        return False

### 3.4 Agent 监控层优化

#### 减少不必要的 Agent 调用

| 任务类型 | 之前 | 之后 | 节省 |
|---------|-----|------|------|
| 数据更新 | Agent | 脚本 | 100% |
| 每日推送 | Agent | 脚本 | 100% |
| 健康检查 | Agent | 脚本 | 100% |
| 监控收集 | Agent | 脚本 | 100% |
| 异常处理 | Agent | Agent | - |

**结果**: 减少 80% 的日常 Agent 调用

#### Agent 介入策略

```yaml
自动处理（脚本）:
  - 确定性任务
  - 定时任务
  - 数据更新
  - 健康检查
  - 监控数据收集

Agent 介入（需要智能决策）:
  - 异常诊断
  - 策略优化
  - 参数调优
  - 根因分析
  - 复杂决策

不介入:
  - 正常运行
  - 预期内的警告
  - 可自动恢复的错误
```
