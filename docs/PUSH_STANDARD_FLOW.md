## 二、数据标准化规范

### 2.1 股票代码格式

**强制要求:** A股代码必须为6位数字

**正确格式:**
- 上海主板: 600000, 600001
- 深圳主板: 000001, 000002
- 创业板: 300001, 300002
- 科创板: 688001, 688002

**错误格式（禁止）:**
- ❌ 302, 306（不完整）

## 三、关键定义与阈值

### 3.1 α因子定义

**α因子得分计算方法:**

```python
α得分 = Σ (因子值 × 因子权重)

# 因子权重（固定）:
- ROE: 30%
- 营收增长: 20%
- 估值因子(PE倒数): 20%
- 动量因子: 15%
- 流动性因子: 15%

# α得分范围: 0-100分
# α得分>80分: 高α，推荐
# α得分<50分: 低α，不推荐
```

**基准:** 相对沪深300指数

### 3.2 风控规则明确定义

**止盈规则:**
- 触发条件: 收益率 > 20%
- 执行方式: 分3批止盈（每批1/3仓位）
  - 第一批: 触发后立即执行（盘中或收盘）
  - 第二批: 次日开盘执行
  - 第三批: 次日收盘执行
- 执行时间: 优先选择收盘前30分钟

**止损规则:**
- 触发条件: 亏损率 < -10%
- 执行方式: 立即全部清仓（强制执行）
- 执行时间: 触发后立即执行（不限时间）
- 滑点容忍: 成交价不超过止损价的±0.5%

**时间触发规则:**
- 触发条件: 持仓天数 > 60天
- 执行方式: 提示评估（非强制）
- 评估内容:
  - α得分变化
  - 基本面变化
  - 行业轮动
- 决策结果: 继续持有/减仓/清仓

**因子触发规则:**
- 触发条件: α得分下降 > 20%（相对建仓时得分）
- 执行方式: 建议换仓（非强制）
- 替代股票: 从当前选股池中选择高α股票
- 执行时间: 次日交易窗口

## 四、交易执行标准操作程序（SOP）

### 4.1 指令接收与核对

**核对清单（每份指令必须通过）:**
- [ ] 股票代码为6位数字，可交易
- [ ] 权重总和 = 100%
- [ ] 买卖方向明确
- [ ] 风控阈值无歧义
- [ ] 执行时间窗口已指定

### 4.2 可执行交易清单生成

**自动化流程:**

```python
# 输入: 总资产金额（如100万元）
# 输出: 可执行交易清单（包含每只股票的计划金额和数量）

def generate_trade_list(portfolio_config, total_asset):
    """
    生成可执行交易清单
    
    Args:
        portfolio_config: 组合配置（核心/卫星/现金权重）
        total_asset: 总资产金额
    
    Returns:
        可执行交易清单DataFrame
    """
    trade_list = []
    
    for stock in portfolio_config['core'] + portfolio_config['satellite']:
        stock_code = stock['code']  # 6位数字
        stock_name = get_stock_name(stock_code)  # 获取股票名称
        weight = stock['weight']
        plan_amount = total_asset * weight
        current_price = get_current_price(stock_code)
        plan_quantity = int(plan_amount / current_price / 100) * 100  # 向下取整到100股
        
        trade_list.append({
            '序号': len(trade_list) + 1,
            '股票名称': stock_name,
            '股票代码': stock_code,
            '方向': '买入',
            '目标权重': f'{weight*100:.1f}%',
            '计划金额': f'{plan_amount:,.0f}元',
            '参考价格': f'{current_price:.2f}元',
            '计划数量': f'{plan_quantity}股',
            '持仓类型': '核心' if stock in portfolio_config['core'] else '卫星'
        })
    
    return pd.DataFrame(trade_list)
```

### 4.3 执行偏差监控

**允许偏差范围:**
- 成交价不超过参考价的±0.5%
- 成交数量不低于计划的95%
- 权重偏差不超过目标权重的±1%

**偏差处理:**
- 偏差 > 允许范围: 立即暂停，报告策略团队
- 偏差 < 允许范围: 记录偏差，继续执行

   - 成交情况

   - 最大回撤
   - α/β暴露

## 七、异常处理机制

### 7.1 数据异常

**情况:** 股票代码格式错误、权重不总和100%

**处理:**
1. 立即停止推送
2. 记录异常日志
3. 发送告警给策略团队
4. 等待修复后重新推送

### 7.2 系统异常

**情况:** 选股失败、因子计算错误

**处理:**
1. 使用备用选股逻辑（简单排序）
2. 明确标注"备用模式"
3. 发送告警给技术团队

### 7.3 市场异常

**情况:** 千股跌停、流动性枯竭

**处理:**
1. 暂停执行所有交易指令
2. 启动应急预案
3. 通知用户紧急情况
